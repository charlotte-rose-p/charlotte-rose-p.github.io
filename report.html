State 3 ways in which your site is accessible.

My site is accessible through the use of alternative text. This allows the information to become available to everyone, as it can be accessed through using a screen reader. The alternative text is also accessible to technologies that cannot see the image, such as search engines.

Having a fixed navigation bar allows the viewer to access the various links at any given time. This means that there is no need to scroll to the top of the page in order to find another section link, it is consequently always available.

My site is easy to use through the clear division of sections with appropriate headings. This enables an ease of navigation for users who may have difficulty in locating different parts of the website. The fonts I have used are also sized appropriately so that the text is readable to everyone.

State 3 ways in which you have considered usability in your site.

I considered usability by focusing on how readable my site is. I achieved this through the input of paragraph formatting and the use of white space to ensure all text can be read easily. I also added transparent boxes upon vibrant backgrounds to enhance readability.

I ensured links within my site to open within a new tab in order to avoid directing users from the main website. This would create confusion and make it difficult to locate back to the original site.

I considered the placement of information to increase the effectiveness of my website. I used columns to distribute information within the page, and insured there to be a different layout for each section. This enables the user to read as much information when scanning the page, as different layouts retain interest. I also positioned important information such as my skills in the centre of the page, so that my strengths can be a focus to professionals.

State 3 things you had to learn/find out by yourself when creating your site, and describe how you did that (i.e. what searches you did, any new tools/techniques you learned, any changes/adaptations you made to a particular resource to make it work with your site).

When creating an image carousel within the Photography section of my site, I researched into how I could manipulate code so that the images would act as a slideshow, meaning the user would not have to select each image. I found these JavaScript functions through the Coursera online course, whereby it detailed how to create a moving image gallery. I adapted this code to function within my own gallery. Through removing the backwards and forwards buttons of my image carousel, I also discovered how to hide elements through the visibility:hidden property which I continued to use within my site. I found this through a search engine which directed me to the visibility property on W3 schools.

I also found out how to change the layout of my nav bar so links were positioned to the left and right, while my nav bar brand remained in the centre to act as a main focus. I researched this through stack overflow- an online community for web programmers. Here I discovered the utility classes of navbar-left, navbar-right and navbar-center, that I applied to my html code.

Furthermore, I had to learn how to create animated skill bars. I was made aware of the basic structure of a skill bar through using the inspect tool on Jordan Vitanov’s personal portfolio, and used html and CSS properties to alter the values and style of the skill bars. I then found the JavaScript properties through research where I discovered codepen- a website that shows editable source code.

What aspect(s) of your work do you think worked well and why?

I think the structure of my website worked well as it is divided clearly into four containers that have different row structures to maintain the interest of users. These are accurately linked through the nav bar which is fixed to be easily accessible. I think my colour scheme works effectively as the highlights of red throughout the site adds unity and draws the user to the most interesting aspects of the page, such as the animated skill bars and buttons to my journalistic work.

What aspect(s) of your work could be improved, and how might you do things differently another time?

My website could be more responsive to mobile devices by being able to adapt all features to a very small screen. This would be done through the concentration of designing for mobile by consistently using @media queries through the construction of my site. If I was to do this again, I would focus more on mobile design from the beginning. My website could also be improved by animations such as the skill bars beginning when the user reaches that position of the page. If I was to do my website differently, I would learn and research greater JavaScript skills prior to producing my website, so I could include these additional features.
